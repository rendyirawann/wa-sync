> **Status:** Berjalan · **Dibuat:** 2026-06-11 · **Pemilik:** rendyirawann
> Peta resmi proyek. Update **§9 Checklist Progres** + **§10 Log Keputusan** tiap selesai langkah.

---

## 0. TL;DR

Rewrite backend **Laravel 12 → Rust**, tampilan **Metronic dipertahankan** (Blade di-port ke **Tera**). Laravel disimpan sebagai **referensi** (tidak di-run). Reuse **DB Postgres `wa_sync`** apa adanya. Tujuan akhir: **WA service platform multi-tenant** (Baileys) — tiap nomor langganan punya sesi WA + dashboard sendiri.

- **Stack:** Axum + SQLx (Postgres) + Tera + tower-sessions + bcrypt. `cargo run` dari `rust/crates/central/` → `http://127.0.0.1:8090`.
- **Beda dari dine-sync:** wa-sync = platform server online multi-tenant, **bukan** local-first POS → tanpa SQLite/sync/offline.
- **Octane?** Tidak relevan (Laravel tak di-run). Axum/Tokio = padanan native Octane: app long-running di memori, pool + template ter-compile, multi-thread async. Tanpa add-on.

## 3. Skema DB terverifikasi (2026-06-11, dari DB live)

17 tabel. Catatan penting untuk implementasi:
- **`users.id` = uuid** (UUIDv7). Hash superadmin `$2y$12$` → bcrypt Laravel, **kompatibel** crate `bcrypt` (verify `$2y$`). Superadmin: `superadmin` / `superadmin@gmail.com`.
- **Spatie:** `roles.id`/`permissions.id` = **bigint**; `model_has_roles.model_id` / `model_has_permissions.model_id` = **uuid**; `role_has_permissions` (perm_id,role_id) bigint. Roles: `Superadmin`(1), `admin`(2), guard `web`. 15 permission (`view_*`, `user.*`, `role.*`), **`category` semua NULL** → grouping M4 fallback.
- **`activity_log`:** `causer_id`/`subject_id` = uuid, `properties` = json. Log names dipakai: login/logout/tambah-edit-hapus user&role/ban/unban/profile/security.
- ⚠️ **`bans`:** `bannable_id`/`created_by_id` = **bigint** (default `morphs`), padahal user UUID → **tidak kompatibel**. Fitur ban lama kemungkinan broken/0 baris. **Rencana M3:** pakai `users.banned_at` sebagai sumber status ban; simpan reason/expiry via kolom tambahan / tabel ban yang dikoreksi (diputuskan di M3, diflag ke user).
- **`sessions`** = format Laravel (tak kompatibel tower-sessions) → pakai store sendiri (MemoryStore dulu / tabel `tower_sessions` Postgres).

## 4b. Gotcha (pelajaran dari dine-sync — hindari)

- Postgres `int4` → sqlx `i32` (cast `::bigint` di SELECT bila perlu i64). `count(*)` sudah bigint.
- sqlx 0.9 `query_as` butuh `&'static str` → const SQL, bukan `format!`.
- Multi-select (roles/permissions) dikirim **CSV via hidden field** (hindari serde_urlencoded repeated-key).
- NUMERIC pecahan → `f64` (`::float8` baca / `$n::numeric` tulis).
- Tera auto-escape (URL slash → `&#x2F;`, tetap valid di browser).
- Postgres `after_connect` → `SET TIME ZONE 'Asia/Jakarta'`.
- Kolom **nullable** via `query_scalar` → tipe ter-infer non-Option (mis. `NaiveDateTime`) → decode NULL gagal (Err). Pakai `query_as` tuple `(Option<T>,)` atau `query_scalar::<_, Option<T>>`.
- `uuid` crate butuh fitur `serde` agar `Uuid: Serialize` (dipakai `serde_json::json!`). AJAX dgn file → multipart (axum fitur `multipart`); tanpa file boleh urlencoded/JSON (cocokkan extractor).
